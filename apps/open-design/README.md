## Build a release

```bash
NODE_PLATFORM=darwin-arm64 ./build-bundle.sh
```

What it does:

- Downloads Node `24.11.0` for `$NODE_PLATFORM` (cached under `.bundle-build/`) and activates pnpm `10.33.2`
  via Corepack, then runs `pnpm install`.
- Builds the daemon (`pnpm --filter @open-design/daemon... build` â†’ `apps/daemon/dist/cli.js`) and the web app
  (`pnpm --filter @open-design/web build` â†’ Next.js static export at `apps/web/out`).
- Stages a self-contained runtime tree under `.bundle-build/stage/`: the daemon (via `pnpm deploy --prod
  --legacy`, including the native `better-sqlite3` binding), `apps/web/out`, the daemon's resource dirs
  (`skills`, `design-systems`, `design-templates`, `craft`, `prompt-templates`, `assets/frames`,
  `assets/community-pets`, `plugins`), and the embedded Node 24 runtime under `runtime/node`.
- Writes `manifest.json` (`kind: node-service`) plus a placeholder `index.html`, normalizes any escaping
  pnpm symlinks (`normalize-symlinks.mjs`), and zips the tree (preserving symlinks).

Output artifact: **`./opendesign-bundle-${NODE_PLATFORM}.zip`** (e.g. `opendesign-bundle-darwin-arm64.zip`).
The script also copies the zip to `/tmp/opendesign-bundle-${NODE_PLATFORM}.zip` (the `localUrl` path the
desktop "Local" install reads) and prints the `sha256` + `sizeBytes` for `apps.js`.

Publish steps:

```bash
# 1. Upload the built zip as a release asset on this repo's tag.
gh release upload opendesign-v1.1.0 ./opendesign-bundle-darwin-arm64.zip

# 2. Update the bundle entry in UnifiedApp apps/desktop/src/data/apps.js with the
#    printed sha256 and sizeBytes (and bump the tag/version if cutting a new release).
```

## Build dependencies

- **Toolchain:** bash, `curl`, `tar`, `zip`, `shasum`, and a `node` on PATH for the `normalize-symlinks.mjs`
  step. Node 24 and pnpm 10.33.2 are **not** required up front â€” the build downloads Node itself and activates
  pnpm through Corepack.
- **Network downloads:** Node `24.11.0` from `nodejs.org` (cached in `.bundle-build/`), and all pnpm workspace
  dependencies from the npm registry on first `pnpm install`.
- **Cross-repo build dependencies: none.** The build downloads its own Node runtime and resolves only
  npm/workspace packages. There are no `../`-relative paths into sibling repos; nothing in the build resolves
  outside this directory, so there is no committed fallback to worry about.
- **Runtime, not build:** the staged daemon talks to **unified-api** at `http://localhost:3141` (overridable
  via `UNIFIED_API_URL`) and mints app-scoped tokens from the UnifiedApp desktop loopback broker. That is a
  runtime dependency on the host, not a build-time one.

## What is NOT committed (regenerated by the build)

- `.bundle-build/` â€” the downloaded Node 24 runtime and the `stage/` tree.
- `node_modules/` (root and workspace) â€” restored by `pnpm install`.
- `apps/daemon/dist/` â€” the built daemon (`cli.js`).
- `apps/web/out/`, `.next/` â€” the Next.js static export and build cache.
- `.turbo/` â€” task cache.
- `opendesign-bundle-*.zip` â€” the release archive (published as a GitHub Release, not committed).

## What we changed vs upstream

- `build-bundle.sh`, `normalize-symlinks.mjs` â€” the bundle build + symlink-containment fixup.
- `apps/daemon/src/unified-auth.ts` â€” loopback-broker token minting (app-scoped JWT; never sees the user's
