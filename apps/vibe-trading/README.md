## Build a release

```bash
# from this app directory (apps/vibe-trading)
# requires: bash, curl, node/npm, zip, and a Python toolchain to pip-install native wheels
NODE_PLATFORM=darwin-arm64 ./build-bundle.sh
```

What it does:

- Downloads + unpacks a relocatable CPython `3.11.10` (python-build-standalone, `install_only`),
  cached under `.bundle-build/`.
- Builds the React/Vite frontend (`npm ci --ignore-scripts && npm run build` → `frontend/dist`) and
  pip-installs `agent/requirements.txt` **into the embedded interpreter** (target-platform wheels).
- Stages the bundle tree (`runtime/python` + `agent/` + `frontend/dist` + `launch.py`), writes
  `manifest.json` (`kind: node-service`, `command: runtime/python/bin/python3 launch.py`, `healthPath: /health`),
  drops a placeholder `index.html`, then runs `normalize-symlinks.mjs` to clamp any symlink that would
  escape the bundle root (the installer rejects the whole archive otherwise).
- Zips (preserving symlinks) and prints the `sha256` + `sizeBytes` for `apps.js`.

> The build must run **on the target platform** — numpy/scipy/pandas/scikit-learn/duckdb wheels are
> platform-specific and cannot be rebuilt on the user's machine. Bump `PBS_RELEASE`/`PY_VERSION` in
> `build-bundle.sh` if the pinned python-build-standalone `install_only` asset 404s for your platform.

**Output artifact:** `./vibe-trading-bundle-darwin-arm64.zip`
(also copied to `/tmp/vibe-trading-bundle-darwin-arm64.zip` for desktop "Local" installs).

Publish steps:

```bash
# 1. Upload the freshly built zip to the matching release tag
gh release upload vibe-trading-v0.1.9 ./vibe-trading-bundle-darwin-arm64.zip --clobber

# 2. Update the UnifiedApp apps.js entry for slug "vibe-trading" with the printed values:
#      sha256:    74cabdb68823078add4cc66aec3e1949451b84e58aa62b5a57dd19d1c83b3073
#      sizeBytes: 338169936
#      version:   0.1.9
```

(If the app is served from a `unified-db` `public.apps`/`public.app_bundles` row, that DB entry
replaces the static `apps.js` entry by slug — keep both in sync if you maintain both.)

## Build dependencies

- **Toolchain (host):** `bash`, `curl`, `node`/`npm`, `zip`, and a Python build toolchain capable of
  `pip install`-ing native wheels for the target platform.
- **Network downloads the build performs:** the relocatable CPython `3.11.10` tarball from
  python-build-standalone (`astral-sh/python-build-standalone`), npm packages for the Vite frontend,
  and the Python wheels in `agent/requirements.txt`.
- **Cross-repo dependencies:** **none.** The bundle is self-contained — it downloads its own CPython
  and embeds it, so there are no `../`-style sibling-repo paths and no committed fallback to maintain.
  (At *runtime*, the launcher talks to the UnifiedAI gateway, defaulting to the desktop's local
  unified-api on `http://localhost:3141`; that is a runtime endpoint, not a build dependency.)

## What is NOT committed (regenerated by the build)

- `.bundle-build/` — the downloaded CPython and the staging tree.
- `node_modules/` (frontend) and `frontend/dist/` (the prebuilt SPA).
- `__pycache__/` and `.pytest_cache/`.
- The release `*.zip` (`vibe-trading-bundle-*.zip`) — published to GitHub Releases, not committed.
- Build logs.

## What we changed vs upstream

**Authored (new) files:**

- `build-bundle.sh` — embeds relocatable CPython, builds the frontend, pip-installs deps into the
  embedded interpreter, stages the tree, writes `manifest.json`, normalizes symlinks, zips.
- `launch.py` — `node-service` entrypoint: binds the host-injected `OD_PORT`, mints an app-scoped
  token from the desktop loopback broker, points the LLM client at unified-api (default
  `http://localhost:3141`), refreshes/self-heals the token (and broker coords on daemon reuse), then
  runs the agent's `api_server`.
- `BUNDLING.md` — the marketplace bundling model + install/test notes.
- `normalize-symlinks.mjs` — clamps any symlink whose target escapes the bundle root.

**Patched upstream files** (adding `VIBE_TRADING_DATA_DIR` data-dir seams + unified-api provider
wiring; defaults unchanged for a plain checkout):

- `agent/api_server.py`
- `agent/src/providers/llm.py`
- `agent/src/providers/llm_providers.json`
- `agent/src/swarm/store.py`
- `agent/tests/test_settings_api.py`
- `frontend/src/lib/api.ts`
- `frontend/src/pages/Settings.tsx`
